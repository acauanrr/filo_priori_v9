Inductive Mode (Original)
  • ❌ Only train TCs as nodes
  • ❌ Test-only TCs are orphans
  • ❌ Orphans get default probability (0.5)
  • ❌ Poor ranking for new tests
  • ❌ Loss of contrast in predictions
Transductive Mode (New)
  • ✔ All TCs (train + test) as nodes
  • ✔ No orphan TCs!
  • ✔ Test TCs get semantic neighbors
  • ✔ Better ranking via GAT propagation
  • ✔ Improved APFD for rare failures

How Transductive Mode Works

In transductive mode, we include ALL test cases (from both train and test sets) as nodes in the phylogenetic graph. Co-failure and co-success edges are only created from training data (where we have labels). Semantic similarity edges are created for ALL nodes, including test-only TCs. This ensures that every test case has at least semantic neighbors, enabling the GAT to propagate meaningful information even for tests without execution history.

Configuration (YAML)
# Enable transductive mode in config graph: build_graph: true use_multi_edge: true edge_types: [co_failure, co_success, semantic] # TRANSDUCTIVE MODE: Include test TCs in graph transductive: true # Eliminates orphan nodes! semantic_top_k: 5 semantic_threshold: 0.75
